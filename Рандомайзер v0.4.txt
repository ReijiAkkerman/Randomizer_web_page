3. Создание категорий списков (личные списки)
    1) Вместо фразы "Списки слов" разместить кнопку выбора категории,
    текстом кооторой будет текущая выбранная категория.
    
    2) При нажатии на кнопку выбора категории появляется выпадающий список 
    уже созданных категорий, в самом конце которого поле для ввода названия 
    новой категории, а справа от поля ввода знак "+" для добавления новой категории.
    
    3) Справа от наименования существующей категории есть значок карандаша - 
    по нажатию на который можно изменить название категории.

    4) Справа от выбранной категории вне ее границ есть кнопка со знаком "+"
    для добавления подкатегории, по нажатию на которую появляется поле для 
    ввода наименования подкатегории. После повторного нажатия на кнопку "+"
    добавляется новая подкатегория и ее наименование появляется на месте где 
    было наименование категории до добавления.

    5) В выпадающем списке категорий по нажатию на кнопку категории:
        5.1) Если категория не содержит подкатегорий - выберется нажатая категория
        5.2) Если категория имеет подкатегории - появится выпадающий список 
        подкатегорий.
        5.3) Если подкатегория также имеет подкатегории - действие 
        будет повторяться рекурсивно, пока не будет выбрана кнопка подкатегории 
        не имеющей подкатегорий.

    6) Категории могут быть созданы в любой из групп.

4. Создание групп списков
    1) Списки могут быть - личные, групповые и от друзей.
    2) Область выбора группы списков находится ниже области основных действий, 
    над областью списков.

    3) Подгруппы создавать нельзя.

5. Сделать возможность делиться списками
    1) Кнопка поделиться списками находится на месте где сейчас кнопка синхронизации 
    данных с github. Кнопка синхронизации с github будет располагаться в разделе 
    настроек, и будет доступна только в локальной версии проекта.

    2) После нажатия на кнопку "поделиться списком" пользователю будет предложено 
    сначала выбрать группу а затем и конкретного пользователя которому список будет
    отправлен. Если необходимо отправить список всем пользователям находящимся 
    в выбранной группе то нужно будет явно выбрать в списке пользователей пункт
    "Всей группе".

    3) Область для выбора группы и затем пользователя будет находиться в том же месте 
    где сейчас открывается область при создании нового списка или списка друдных слов.
    
    4) После отправки списка пользователю - получатель получит уведомление о 
    небходимости принять или отклонить предлагаемый список, а у отправителя выбранный
    для отправки список появится в группе отправленные - выделенный оранжевым цветом 
    говорящий пользователю о том что получатель еще в раздумьях принять список или 
    отклонить.

        4.1) Если список был отправлен более чем одному пользователю то список
        в группе отправленные будет отображаться оранжевым цветом пока все получатели 
        не дадут ответ в отношении этого списка.

        4.2) Если мнение пользователей в отношении списка расходится (например 
        Вася принял список, а Петя его отклонил) - то у отправителя список выделяется
        пурпурным цветом.

        4.3) Если все пользователи приняли список - он выделяется зеленым цветом.
        4.4) Если все пользователи отклонили список - его цвет становится красным.
        4.5) Если список был отправлен одному пользователю и он его принял - 
        цвет списка у отправителя становится зеленым.

        4.6) Если список был отправлен одному пользователю и он его отклонил - 
        цвет списка у отправителя становится красным.

        4.7) Если список был отправлен нескольким пользователям - то для просмотра 
        статуса списка по каждому из получателей справа от наименования списка будет 
        находиться кнопка статистики со соответствующим значком, по нажатию на которую 
        будет открываться список получателей списка выделенных соответствующим 
        их ответу цветом. Т.е. оранжевым выделяются получатели не принявшие решение,
        зеленым цветом принявшие список, а красным отклонившие его.
    
    5) Полученные списки от других пользователей сразу будут попадать в группу 
    "Полученные". Списки в отношении которых пользователь не принял решение в группе 
    "Полученные" будут также выделены оранжевым цветом. По аналогии описанной выше 
    зеленый - значит принят, красный - отклонен.

    6) Ответить сразу на набор списков нельзя. Давать ответ обязательно по каждому 
    списку отдельно.

    7) После принятия списков их группу можно сменить. Для этого справа от наименования
    списка будет кнопка перемещения с соответствующим значком, по нажатию на которую 
    ниже области основных действий появляется кнопка выбора группы и кнопка выбора
    категории (если категорий нет или нет нужной можно прямо там создать новую). 
    ниже этих двух кнопок есть кнопка "Переместить".

    8) После перемещения списка, из группы "Полученные" по умолчанию он пропадает - 
    однако для просмотра истории полученных списков в настройках можно поставить галочку 
    на отображение списков ответ по которым был дан.

    9) Если для группы "Отправленные" в настройках стоит галочка на скрытие списков
    по которым дан ответ - то списки скрываются из группы, но могут быть отображены
    если эту галочку убрать.

    10) Получение списков от заблокированных профилей невозможно.
